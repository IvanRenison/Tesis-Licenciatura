El primer \texttt{for} de \texttt{add\_poly} asÃ­ como estÃ¡ casi que se podrÃ­a paralelizar, lo Ãºnico que lo impide es que \texttt{add\_amb} hace un \texttt{ambs.push} que serÃ­a problemÃ¡tico si llega a ejecutar al mismo tiempo por mÃ¡s de un hilo. Para solucionar eso lo que se hizo es guardar las ambigÃ¼edades en varios vectores, uno por cada polinomio de \texttt{G} y despuÃ©s al final de la funciÃ³n agregar todas las ambigÃ¼edades a \texttt{ambs}.         size_t d = amb.size();
        while (ambs_per_deg.size() <= d) {
          ambs_per_deg.push_back({});
        }
        ambs_per_deg[d].push_back({amb, i, j});
      }
    }
  }
\end{minted}

Con esto el primer \texttt{for} se puede paralelizar directamente.
